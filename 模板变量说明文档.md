# 学情分析模板变量说明文档

> 所有变量均使用单花括号 `{}` 包裹占位符，在模板中写入 `{变量名}` 即可。
> 变量命名统一使用**驼峰格式**（如 `{taskTitle}`），旧点号格式（如 `{task.title}`）仍兼容但不再推荐。
> 若模板中引用的变量在当前分析场景下无数据，系统将自动替换为空字符串，不会导致错误。

---

## 一、QUIZ_DESIGN（课堂作业设计模板）

| 变量名 | 变量含义及作用 | 内容示例 |
|--------|--------------|---------|
| `{taskTitle}` 课题 | 课题名称，用于 AI 设计作业时了解课堂主题 | 分数的意义 |
| `{taskGrade}` 年级 | 年级，用于适配不同年龄段学生的作业难度 | 三年级 |
| `{taskSubject}` 学科 | 学科，用于确定作业的知识范围 | 数学 |
| `{taskObjectives}` 课堂目标 | 课堂目标，明确本节课的教学目标 | 理解分数的含义，掌握真分数与假分数的概念 |
| `{taskDescription}` 课堂目标 | 课堂目标（同 taskObjectives），所属课堂的教学目标 | 理解分数的基本概念 |
| `{kbContent}` 知识库内容 | 知识库内容，提供给 AI 参考的背景知识 | 分数的基本定义、分数与除法的关系 |
| `{kbList}` 参考知识库列表 | 知识库列表，格式化后的多个知识库内容 | 【分数基础】：分数的定义……【分数进阶】：…… |
| `{convDescription}` 对话目标 | 对话目标，对话活动预设的目标信息 | 通过对话帮助学生理解分数的基本概念 |
| `{teacherName}` 教师姓名 | 教师姓名，可用于作业开头或结尾的署名 | 王老师 |
| `{quizCount}` 出题数量 | 出题数量，AI 生成作业的题目数量 | 5 |

## 附录

### 变量命名规范

- **统一使用驼峰格式**：`{taskTitle}`, `{taskGrade}`, `{quizTitle}`, `{explorationTitle}`
- **旧点号格式兼容**：`{task.title}`, `{quiz.title}`, `{exploration.title}` 仍可使用，但不再推荐
- **同义变量**：`{taskDescription}` 和 `{taskObjectives}` 含义相同，均表示"课堂目标"

### 变量层级速查

| 层级 | 模板类型 | 推荐作业变量 |
|------|---------|------------|
| 对话活动层（PC） | student | `{personalDialogContents}` 个人对话记录 |
| 课堂层（Task） | student | `{personalDialogContents}` 个人对话记录 + `{personalQuizStats}` 个人作业数据 |
| 课堂层（Task） | class | `{classQuizStats}` 全班作业数据 + `{classDialogAnalysisReport}` 班级对话分析报告 |

### 重要说明

- `{personalDialogAnalysisReport}` 个人对话分析报告 和 `{classDialogAnalysisReport}` 班级对话分析报告 只有在**开启"使用下级分析"**（TASK_INSIGHTS）时才有数据，否则为空
- 课堂层全班分析的 `{personalDialogContents}` 个人对话记录 包含该班所有学生在该课堂所有对话活动中的对话记录
- 各题正确率中 "基" = 基础题，"进" = 提升题
- `{personalQuizStats}` 个人作业数据 在 class 类型模板中为全班作业明细；在 student 类型模板中为该学生个人明细
