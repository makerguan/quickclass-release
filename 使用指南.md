# QuickClass 使用指南

## 1. 系统概述

QuickClass 是一个 AI 驱动的课堂教学辅助平台，帮助教师创建互动式学习活动、管理课堂作业、分析学情数据、生成教研论文与课题方案。核心功能包括：

- **对话活动**：学生与 AI 进行主题对话，深入理解知识点
- **课堂作业**：在线答题，自动批改，即时反馈（**支持单选、多选、填空、简答**，多选题自动标注 `[多选]`）
- **互动探究**：HTML 交互式学习活动，支持提交、评分、行为分析和 **AI 伴学**（学生困惑时 AI 实时引导）
- **学情分析**：AI 自动生成个人和全班分析报告（支持文本型和可视化HTML两种形式）
- **提示词模板系统**：六类可自定义模板，教师可控制 AI 分析的维度和输出格式
- **知识库**：上传教学资料，AI 对话时自动引用
- **🆕 教学研究（教研宝）**：基于课堂真实数据，AI 生成 **教学论文** 与 **课题方案**，并支持**下载为 Word 文档**（含研究框架图）

### 1.1 适用场景

- 课堂教学中的个性化辅导
- 课后作业布置与自动批改
- 学情数据采集与多维分析
- 探究式学习活动设计
- 教学反思与教研沉淀
- **🆕 教科研论文与课题方案撰写**（教研宝）

### 1.2 教学创新理念

QuickClass 倡导 **"AI 赋能教师，而非替代教师"** 的协同范式：

| 角色 | 职责 |
|------|------|
| **教师** | 设计学习目标、引导策略、分析维度、教学决策、教研方向 |
| **AI** | 执行个性化引导、自动批改、行为分析、报告生成、教研论文撰写辅助 |

## 2. 环境要求

在启动 QuickClass 之前，请确保**教师机**（安装服务端的那台电脑）已安装以下软件：

### 2.1 Node.js（必须）

| 项目 | 要求 |
|------|------|
| 版本 | **Node.js 18 或更高版本** |
| 下载地址 | https://nodejs.org/ |
| 推荐版本 | LTS（长期支持版） |

#### 安装步骤

1. 访问 https://nodejs.org/
2. 下载对应系统的 LTS 版本安装包
   - **Windows**：下载 `.msi` 安装包
   - **macOS**：下载 `.pkg` 安装包
3. 双击安装包，按提示完成安装（保持默认选项即可）
4. 安装完成后，**重启命令行/终端**

#### 验证安装

打开命令行（Windows `cmd` 或 macOS `终端`），执行：

```bash
node -v
npm -v
```

正确显示版本号（如 `v20.10.0`）即表示安装成功：

```
$ node -v
v20.10.0

$ npm -v
10.2.3
```

> **注意**：启动脚本会自动检测 Node.js 版本，如果未安装或版本过低，会给出明确提示。

### 2.2 网络要求

| 场景 | 要求 |
|------|------|
| 教师机访问 | 浏览器访问 `http://localhost:3000` |
| 学生访问 | 同一局域网内，浏览器访问 `http://教师机IP:3000` |
| AI 服务 | 需联网调用 AI 接口（除非使用本地 AI 服务） |
| 🆕 教研宝 | 同上，需要 AI 服务生成论文/课题 |

#### 查看教师机 IP 地址

启动脚本会自动检测并显示本机所有可用的 IP 地址，例如：

```
教师访问地址：http://localhost:3000

学生访问地址（局域网）：
  http://192.168.1.100:3000
  http://192.168.1.101:3000
```

#### 防火墙配置（学生无法访问时）

如果学生无法通过 IP 访问教师机，需在教师机上配置防火墙：

- **Windows 防火墙**：允许 `3000` 端口入站
  1. 打开"Windows Defender 防火墙高级设置"
  2. 入站规则 → 新建规则 → 端口 → TCP → 特定本地端口 3000
  3. 允许连接 → 完成
- **macOS 防火墙**：系统设置 → 网络 → 防火墙 → 允许 Node.js 接受传入连接

### 2.3 AI 服务（可选但推荐）

QuickClass 需要调用 AI 服务才能生成对话回复、批改作业、分析学情，**以及生成教研论文/课题方案**。

#### 推荐服务

| 服务商 | 优势 | 申请地址 |
|--------|------|----------|
| 阿里云通义千问 | 国内访问快，有免费额度 | https://dashscope.console.aliyun.com/ |
| DeepSeek | 性价比高，推理能力强 | https://platform.deepseek.com/ |
| 智谱 AI | GLM 系列，中文优秀 | https://open.bigmodel.cn/ |
| OpenAI | GPT-4 效果最好，需科学上网 | https://platform.openai.com/ |

> 系统支持**所有 OpenAI 兼容接口**的服务。申请 API Key 后，在教师端的「系统设置 → AI 服务配置」中填入即可。

## 3. 快速启动

### 3.1 启动服务

#### Windows

双击 `start-windows.bat`，脚本会自动：
1. 检查 Node.js 环境
2. 安装依赖（如未安装）—— 包括 `sharp`、`katex`、`docx` 等
3. 生成 Prisma 客户端
4. **🆕 首次启动自动初始化数据库**（无需预置数据库文件）
5. 构建生产版本（首次运行约 1-3 分钟）
6. 启动服务并显示**局域网访问地址**

#### macOS

```bash
bash start-mac.sh
```

#### 访问地址

启动成功后，脚本会显示：

```
教师访问地址：http://localhost:3000
学生访问地址（局域网）：
  http://192.168.1.100:3000
  http://192.168.1.101:3000
```

> 服务默认监听 `0.0.0.0`，学生可通过局域网 IP 访问。

### 3.2 🆕 首次启动与自动 Seed

本版本采用**全新用户体验**：

1. **首次访问**：自动显示教师注册页面
2. **用户填写信息并提交**：系统创建第一个教师账号
3. **🆕 自动填充演示数据**（约 1-3 秒）：
   - 2 个示例班级（邀请码：`MATH2026`、`MATH2026B`）
   - 多个示例学生（含姓名、座右铭）
   - 1 个示例课堂（含完整对话活动、作业、互动探究、预设对话）
   - 真实示例学生对话（5-10 条消息）
   - 示例课堂作业（多套，含学生作答记录）
   - 示例互动探究活动
   - 完整提示词模板（六类，含"五类分析模板 + 对话设计"）
   - 默认系统配置（AI 模型、并发限制等）

> **设计目的**：方便教师**上手即看到所有功能演示**，无需手动造数据；适合培训、测试、体验场景。

### 3.3 停止服务

- **Windows**：双击 `stop.bat`
- **macOS**：运行 `bash stop.sh` 或按 `Ctrl+C`

## 4. 教师端

### 4.1 首次配置

#### 首次启动的用户设置

首次启动时，浏览器会自动打开 `http://localhost:3000`，显示用户配置页面：

1. 填写**手机号**、**邮箱**、**姓名**、**学校/机构**
2. 设置**密码**（至少 6 位）和**确认密码**
3. 可选：设置**密码恢复问题**和答案
4. 点击「完成用户设置」

> **提示**：学校/机构名称需包含"学"、"园"、"院"、"中心"等关键词。

#### 密钥文件保存

注册成功后，系统会弹出**密钥文件**保存窗口：

- 密钥用于**忘记密码时恢复账号**
- **只能下载一次**，请务必妥善保管
- 点击「下载密钥文件」保存到本地
- 点击「已保存」进入教师工作台

#### 🆕 演示数据自动填充

完成注册后，系统会在后台**异步**填充演示数据（参见 §3.2）。稍等 1-3 秒，进入工作台即可看到完整的示例数据。

#### 后续登录

已注册用户访问 `http://localhost:3000` 时，会自动跳转到登录页：

1. 输入手机号和密码
2. 点击「登录」
3. 登录后自动进入教师工作台

#### 忘记密码

在登录页面点击「忘记密码」，按提示通过**密保问题**重置密码。

### 4.2 创建课堂与班级管理

#### 创建课堂

1. 进入「课堂管理」页面
2. 点击「新建课堂」
3. 填写课堂信息：
   - **课堂标题**（必填）：如「一元一次方程」
   - **课堂目标**（必填）：描述本课堂的学习目标
   - **年级**（必填）：选择适用年级
   - **学科**（必填）：选择学科
   - **班级**（必填）：选择要分配的班级
4. 点击「保存」

> 🆕 **课堂管理 UI 优化**：课堂列表支持**展开/折叠**、**批量操作**、**快速预览**功能，教师可一目了然看到所有课堂的活动汇总。

创建成功后，课堂自动展开并进入对话活动编辑模式。

#### 班级管理

1. 进入「班级管理」页面
2. 可创建新班级，每个班级自动生成**邀请码**
3. 学生通过邀请码加入班级
4. 可设置是否开放邀请码注册（开启后学生可自助注册）

#### 课堂状态

| 状态 | 说明 |
|------|------|
| 已停用 | 课堂未发布，学生不可见 |
| 已启用 | 课堂已发布，学生可见可使用 |
| 已归档 | 课堂已结束，学生不可见 |

### 4.3 对话活动管理

对话活动是学生与 AI 进行主题对话的核心载体。

#### 编辑对话活动

1. 在课堂列表中展开目标课堂
2. 点击「活动管理」进入编辑模式
3. 可添加、编辑、删除对话活动

#### 对话活动字段说明

| 字段 | 说明 | 是否必填 |
|------|------|----------|
| 对话活动名称 | 活动标题，如「什么是方程？」 | 是 |
| 对话活动目标 | 活动描述和学习目标 | 是 |
| 对话提示词 | 注入 AI 的系统提示词，决定 AI 的角色和行为 | 是 |
| 个人学情分析提示词 | 生成学生个人分析时使用的提示词 | 是 |
| 全班学情分析提示词 | 生成全班分析时使用的提示词 | 是 |

#### AI 角色设计建议

对话提示词是 AI 在对话中的"行为准则"，建议遵循以下原则：

- **以问代答**：多通过提问引导学生思考，而非直接给出答案
- **鼓励质疑**：欢迎学生提出不同观点，培养批判性思维
- **联系生活**：适当联系学生熟悉的生活实例，帮助理解抽象概念
- **适时总结**：在学生理解关键知识点后，进行简明扼要的总结
- **分层引导**：根据学生的回答深度调整问题难度

#### 启用/禁用对话活动

- 在课堂列表中，每个对话活动旁有开关按钮
- 禁用后学生端不再显示该对话活动
- 已有的学生对话记录不会删除

#### 排序

- 使用上下箭头按钮调整对话活动顺序
- 学生端按此顺序显示

### 4.4 课堂作业管理

#### 创建作业

1. 在课堂展开区域，点击「作业管理」
2. 点击「新建作业」
3. 填写作业标题和描述

#### 添加题目

1. 进入作业详情，点击「添加题目」
2. 选择题型：**单选题**、**🆕 多选题（自动标注 `[多选]`）**、填空题、简答题
3. 填写题目内容、选项（选择题）、答案、分值、难度
4. 也可使用 **AI 自动生成题目**

> 🆕 **多选题清晰区分**：教师添加多选题时，系统会在题号前自动加 `[多选]` 标签（如「1. [多选] 下列哪些是...」），避免学生混淆。AI 批改时按多选计分规则（多选漏选、错选均不给分）。

#### 作业状态

| 状态 | 说明 |
|------|------|
| 未生效 | 草稿状态，学生不可见。编辑保存后自动回到此状态 |
| 已生效 | 已发布，学生可答题。需至少添加一道题目才能生效 |

> **注意**：每次编辑保存作业后，状态会自动变为「未生效」，需重新发布。

#### 作业统计与报告

- **统计**：查看答题人数、平均分、正确率（含每题正确率柱状图）
- **报告**：AI 自动生成作业分析报告（详见§5.6）

### 4.5 互动探究管理

互动探究是 HTML 交互式学习活动，支持学生操作、提交、行为分析和 AI 伴学。

#### 创建探究活动

1. 在课堂展开区域，点击「互动探究」面板
2. 点击「新建探究」
3. 填写标题、描述
4. 编辑 HTML 交互内容（支持直接编辑 HTML 代码）

#### AI 生成题目

1. 编辑探究内容后，点击「AI 生成题目」
2. 系统根据探究内容自动生成 4 道题目（1 单选 + 1 多选 + 1 填空 + 1 简答）
3. 生成后可手动调整

> **注意**：已有学生提交后，不可修改探究内容和重新生成题目。

#### 提交功能

- 开启后学生可提交探究结果
- 已有提交后，内容和提交开关将被锁定

#### 行为分析

系统会自动记录学生在探究活动中的：
- 各类操作（点击、拖拽、输入）的频次与占比
- 提交时停留时长
- 答题尝试次数
- 互动行为模式

这些数据将用于学情分析报告。

#### 🆕 AI 伴学（学生端）

互动探究可**单独启用 AI 伴学**功能，开启后：

- 学生遇到困惑时，可与 AI 进行**辅助对话**
- AI 根据探究主题给予**引导性提示**（不直接给答案）
- AI 伴学消息会被教师端查看，便于了解学生思维过程

**教师端开启方式**：
1. 在互动探究详情面板
2. 勾选「**启用 AI 伴学**」
3. 保存即可

> 🐞 **修复记录**：早期版本 AI 伴学开启后，学生消息能显示但 AI 不回复（一直显示"正在输入..."）。**该问题已在最新版本修复**——AI 现在能正常返回引导语，可放心启用。

## 5. 学情分析（核心功能）

学情分析是 QuickClass 的核心功能。系统基于学生的对话记录、作业答题和探究提交数据，由 AI 自动生成多维度分析报告。

### 5.1 学情分析矩阵

QuickClass 首创**"三类活动 × 两层视角"**的学情分析体系：

| 活动维度 | 学生个人分析 | 全班班级分析 |
|----------|--------------|--------------|
| **对话活动** | 个体对话表现分析 | 班级对话汇总分析 |
| **互动探究** | 个人探究行为分析 | 班级探究行为分析 |
| **课堂作业** | 个人答题情况分析 | 班级答题数据分析 |
| **综合课堂** | 学生个人综合分析 | 全班综合分析 |

**价值**：
- 横向看：从**活动维度**对比（哪个活动最有效）
- 纵向看：从**学生维度**追踪（某学生各活动表现）
- 整体看：从**班级维度**把握（找共性问题）

### 5.2 双报告模式

每类分析支持**两种输出形式**：

| 报告类型 | 特点 | 适用场景 |
|----------|------|----------|
| **文本型报告** | 精炼文字，结构清晰 | 快速浏览、复制粘贴、邮件发送 |
| **数据型（HTML）报告** | 含 ECharts 可视化图表（雷达图、柱状图、饼图、散点图） | 深度分析、投屏展示、打印存档 |

教师可在分析界面自由选择使用哪种报告。

> 🆕 **AI 报告模板选择优化**：分析页面的模板下拉菜单更加清晰，教师可一眼识别「文本型 vs 数据型」、「系统预设 vs 我的模板」，减少误选。

### 5.3 进入学情分析

1. 在课堂列表中，点击课堂旁的「分析」按钮
2. 或进入「课堂管理」→ 展开课堂 → 点击具体活动的「分析」

---

### 5.4 对话活动分析

基于学生与 AI 的对话记录，分析学生对知识的理解程度。

#### 学生个人分析

## 6. 🆕 教学研究（教研宝）

教研宝是 QuickClass 最新发布的**教学研究辅助模块**，基于真实的课堂数据（对话、作业、探究），由 AI 自动撰写教学论文与课题方案。

### 6.1 进入教研宝

在工作台左侧菜单，点击「**教学研究**」进入。

### 6.2 创建研究项目

1. 点击「**新建研究项目**」
2. 填写项目信息：
   - **项目名称**（必填）
   - **项目类型**（必选）：
     - **教学论文**：学术性研究文章，6 要素或 6 章节结构
     - **课题方案**：申报书风格，8 章节，含研究框架图
3. 选择**关联课堂**（至少 1 个）
4. 选择**数据类型**：
   - ✅ 对话数据
   - ✅ 作业数据
   - ✅ 探究数据
   - 可全选以获得最丰富的素材

### 6.3 🆕 生成候选题目

点击「**生成候选题目**」，AI 根据所选课堂数据生成 **10 个候选题目**：

#### 教学论文类题目特点

| 范围 | 题目类型 | 标志 |
|------|----------|------|
| 前 5 个 | 🔬 实践研究类（实证数据驱动） | 带"实践研究"标签 |
| 后 5 个 | 📖 案例分析类（典型案例研究） | 带"案例分析"标签 |

#### 课题方案类题目特点

10 个题目**覆盖 9 种研究方法**：
- 行动研究
- 案例分析
- 调查研究
- 实验研究
- 准实验
- 叙事研究
- 内容分析
- 设计本位
- 混合方法

### 6.4 🆕 生成论文/课题初稿

选中 1 个题目后，点击「**生成论文/课题初稿**」，AI 自动撰写完整内容。

#### 教学论文结构

| 类型 | 结构 |
|------|------|
| 实践研究 | 6 要素结构：问题提出 → 研究设计 → 数据分析 → 实践反思 → 教学建议 → 参考文献 |
| 案例分析 | 6 章节结构：背景 → 案例描述 → 分析框架 → 案例剖析 → 启示 → 参考文献 |

#### 课题方案结构（8 章节）

1. 问题的提出
2. 研究的意义
3. 研究的依据与理论基础
4. 研究的思路、过程与方法（**🆕 含研究框架图**）
5. 研究对象与内容
6. 研究预期成果与可能的创新点
7. 研究的基础与保障条件
8. 进度安排与参考文献

### 6.5 🆕 研究框架图

课题方案自动附带**研究框架图**，特点：

- **结构化 JSON** 输入（AI 自动嵌入 `[FRAMEWORK_JSON_START]...` 段）
- **4 级灰度配色**，无大块纯黑
- **统一白底大方框**，全部内容在框内
- **渲染为 PNG 高分辨率图**（200 DPI），可直接嵌入 Word
- 自动呈现 **研究阶段 → 研究内容 → 研究方法** 三列逻辑

> 技术细节：使用 `sharp` 库将 SVG 转为 PNG，所以教研宝依赖 `sharp` 包（已在 `package.json` 中）。

### 6.6 🆕 AI 决定图示占位

论文/课题中的某些位置（如数据可视化、教学示例），AI 可以自动插入「**图示占位**」（如 `<!-- 图：XXX -->`），便于后续教师手动或二次 AI 填充。

### 6.7 选题与下载

1. 在生成的内容页面，点击「**下载 Word 文档**」
2. 系统使用 `docx` 库直接生成 `.docx` 文件
3. 自动包含：
   - 标题、摘要、关键词
   - 正文（含章节、列表、表格）
   - 研究框架图（如有）
   - 参考文献

### 6.8 典型工作流

```
新建项目 → 选择课堂 → 选择数据类型 → 生成 10 个候选题目
   ↓
选中一个题目 → 生成初稿（5-15秒） → 检查内容
   ↓
（可选）多次重新生成 → 选中最佳版本
   ↓
下载 Word → 微调 → 投稿/提交课题申报
```

### 6.9 教研宝的最佳实践

- **确保数据充分**：选择至少 2-3 个有对话/作业数据的课堂，AI 才能写出有血有肉的内容
- **多次生成对比**：同一题目，AI 多次生成的内容风格各异，建议生成 2-3 次后人工挑选最佳
- **题海策略**：先看 10 个题目的方向，再决定是否重新生成
- **Word 微调**：AI 写的只是初稿，下载后建议二次编辑，加入最新理论和真实案例
- **保存多个版本**：所有生成的内容都自动保存在数据库，可随时回看

## 8. 常见问题

### Q: 学生看不到课堂？

检查以下几点：
1. 课堂状态是否为「已启用」
2. 课堂是否分配到了学生所在的班级
3. 对话活动是否已启用

### Q: AI 对话无响应？

1. 检查系统设置中的 AI 服务配置是否正确
2. 点击「测试连接」验证 AI 服务可用性
3. 检查 API Key 是否有效
4. 检查网络连接

### Q: 互动探究 AI 伴学不回复？

**该问题已在最新版本修复**。如仍出现：
1. 确认教师端已勾选「启用 AI 伴学」
2. 升级到最新版本（参见本指南 §9 版本更新要点）

### Q: 课堂作业多选题算分不对？

本系统多选题按以下规则计分：
- 全部正确 → 得满分
- 部分正确但漏选 → 部分得分
- 选错任何一项 → 不得分

如果仍不对，请检查答案设置。

### Q: 🆕 教研宝生成失败？

1. 检查是否有可用的课堂数据（对话/作业/探究）
2. 检查 AI 服务（教研宝与对话共用配置）
3. 检查所选课堂**至少有 1 个学生**有数据
4. 论文生成需要 10-30 秒，请耐心等待

### Q: 🆕 教研宝下载的 Word 没图？

研究框架图自动嵌入。如果缺失：
1. 确认内容中包含 `[FRAMEWORK_JSON_START]...` 段
2. 检查服务器日志，看是否有 "framework-diagram 生成失败"
3. 重新生成初稿

### Q: 🆕 首次启动数据库创建后没有数据？

确认您提交了教师注册表单。**演示数据只在第一个教师注册成功后自动填充**。如果误删了数据库，可重新跑：

```bash
npx prisma db push --skip-generate --accept-data-loss
npm run db:seed
```

### Q: 学情分析生成失败？

1. 确保学生已有对话记录或答题数据
2. 检查 AI 服务是否正常
3. 检查分析提示词是否为空
4. 确认已选择有效的分析模板

### Q: 如何切换文本型/数据型报告？

在分析页面，分析时使用的模板决定报告类型：
- 文本型模板 → 生成 Markdown 文本报告
- 数据型（HTML）模板 → 生成可视化 HTML 报告

可在「提示词模板」管理页面查看每个模板的输出类型。

### Q: 保存对话活动后，之前的对话记录丢失关联？

此问题已在最新版本中修复。保存对话活动时，系统会自动保留学生对话记录与对话活动的关联。

### Q: 如何重置数据库？

```bash
npx prisma db push --force-reset
npx tsx prisma/seed-full.ts
```

> **警告**：此操作会删除所有数据，不可恢复！先备份数据库文件 `prisma/dev.db`。

### Q: 如何更新软件版本？

本软件提供了一键更新脚本，**自动备份数据库 + 同步新版代码 + 恢复数据库 + 迁移数据库结构 + 重新构建**，5 步全自动完成。

#### ⚠️ 更新前准备

| 备份项 | 位置 | 说明 |
|--------|------|------|
| 数据库 | `prisma/dev.db` | 包含所有班级、学生、对话、作业、教研项目等数据 |
| 自定义模板 | 数据库内 | 模板存储在数据库中，随 dev.db 一起保留 |
| `.env` 文件 | 项目根目录 | 包含 API Key 等配置（**更新时会被覆盖**） |

#### 🔄 自动更新流程（推荐）

**Windows**：
1. 将新版「跨平台安装包」文件夹解压到**旧版目录的同级**
2. 双击旧版目录中的 `update-windows.bat`
3. 等待脚本执行（备份数据库 → 同步新版文件 → 恢复数据库 → 迁移 → 重新构建）
4. 完成后用 `start-windows.bat` 重新启动

**macOS**：
1. 将新版「跨平台安装包」文件夹解压到**旧版目录的同级**
2. 在旧版目录执行：
   ```bash
   bash update-mac.sh
   ```
3. 等待脚本执行
4. 完成后用 `start-mac.sh` 重新启动

#### 📋 脚本自动执行的 5 个步骤

| # | 步骤 | 说明 |
|---|------|------|
| 1 | 备份数据库 | `dev.db` → `dev.db.bak` |
| 2 | 同步新版文件 | 排除数据库、缓存、依赖 |
| 3 | 恢复数据库 | `dev.db.bak` → `dev.db` |
| 4 | 更新数据库结构 | `prisma migrate deploy`（安全迁移，**不丢数据**） |
| 5 | 重新构建 | `npm run build` |

#### 📦 数据保留保证

- ✅ **不丢学生数据**：数据库完全保留
- ✅ **不丢教师数据**：所有教师账号、班级、学习任务全部保留
- ✅ **不丢教研项目**：所有生成的论文、课题方案都保留
- ✅ **不丢自定义模板**：六类提示词模板都保留
- ✅ **不丢 AI 伴学对话**：所有互动探究的 AI 伴学消息都保留

#### ❓ 常见更新问题

**Q：更新后数据会丢失吗？**
A：不会。脚本会先备份 + 后恢复，`prisma migrate deploy` 只升级结构，**不删数据**。

**Q：更新后启动报错？**
1. 确认 `prisma/dev.db` 文件存在
2. 手动执行：```bash
   npx prisma generate
   npx prisma migrate deploy
   npm run build
   ```
3. 检查 `.env` 文件配置

**Q：如何回退到旧版本？**
1. 重新解压旧版 zip
2. 恢复 `prisma/dev.db` 备份（如有）
3. 执行 `npm install && npx prisma generate && npm run build`

**Q：`.env` 文件被覆盖怎么办？**
新版 `.env` 可能包含新配置项。对比新旧 `.env`，把你之前的 API Key 等填入新版。

**Q：`prisma migrate deploy` 失败？**
可能是数据库结构与迁移记录不一致：
```bash
npx prisma db push --skip-generate --accept-data-loss
```
> ⚠️ 这个命令可修复但可能在极小概率丢数据，请先确保已备份 `dev.db`。

## 9. 教学创新特点

QuickClass 的核心创新可以概括为：

> **从"经验驱动教学"到"数据驱动教学"**
> **从"一对多讲授"到"一人一案引导"**
> **从"结果评价"到"过程行为分析"**
> **从"AI 替代教师"到"AI 赋能教师"**
> **🆕 从"经验式教研"到"数据驱动教研"**

### 9.1 解决的核心痛点

- **个性化教学难以规模化**：AI 对话 + 自动批改，让每个学生都有"专属辅导"
- **作业批改与反馈滞后**：自动批改 + 即时反馈
- **学情分析缺乏数据支撑**：多维度自动分析，结构化报告
- **课堂互动形式单一**：对话活动 + 互动探究，主动参与
- **教学反思缺乏工具**：自动生成报告，可回溯可对比
- **🆕 教研写作负担重**：基于真实数据 AI 自动撰写初稿，教师微调即可用

### 9.2 全国首创特点

1. **"三类活动 × 两层视角"学情分析矩阵**：横纵结合，全面把握
2. **"文本型 + 数据型"双报告模式**：同一任务，两种输出
3. **模板驱动的 AI 分析系统**：教师掌控分析逻辑
4. **"AI 引导者"角色定位**：培养思维，而非获取答案
5. **从"结果分析"到"行为分析"**：理解"怎么知道"，而非只知"知不知道"
6. **"教师设计 + AI 执行"协同范式**：保留教师主导权
7. **🆕 互动探究 AI 伴学**：学生困惑实时引导，思维过程可追溯
8. **🆕 教学研究自动化**：基于课堂数据一键生成论文与课题方案
9. **🆕 教研宝下载为 Word**：含研究框架图，可直接投稿
10. **完整本地化部署能力**：数据安全可控
