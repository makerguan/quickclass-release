# QuickClass 使用指南

## 1. 系统概述

QuickClass 是一个 AI 驱动的课堂教学辅助平台，帮助教师创建互动式学习活动、管理课堂作业、分析学情数据。核心功能包括：

- **对话活动**：学生与 AI 进行主题对话，深入理解知识点
- **课堂作业**：在线答题，自动批改，即时反馈
- **互动探究**：HTML 交互式学习活动，支持提交、评分和行为分析
- **学情分析**：AI 自动生成个人和全班分析报告（支持文本型和可视化HTML两种形式）
- **提示词模板系统**：六类可自定义模板，教师可控制 AI 分析的维度和输出格式
- **知识库**：上传教学资料，AI 对话时自动引用

### 1.1 适用场景

- 课堂教学中的个性化辅导
- 课后作业布置与自动批改
- 学情数据采集与多维分析
- 探究式学习活动设计
- 教学反思与教研沉淀

### 1.2 教学创新理念

QuickClass 倡导 **"AI 赋能教师，而非替代教师"** 的协同范式：

| 角色 | 职责 |
|------|------|
| **教师** | 设计学习目标、引导策略、分析维度、教学决策 |
| **AI** | 执行个性化引导、自动批改、行为分析、报告生成 |

## 2. 快速开始

### 2.1 启动服务

#### Windows

双击 `start-windows.bat`，脚本会自动：
1. 检查 Node.js 环境
2. 安装依赖（如未安装）
3. 生成 Prisma 客户端
4. 构建生产版本（首次运行约 1-3 分钟）
5. 启动服务并显示**局域网访问地址**

#### macOS

```bash
bash start-mac.sh
```

#### 访问地址

启动成功后，脚本会显示：

```
教师访问地址：http://localhost:3000
学生访问地址（局域网）：
  http://192.168.1.100:3000
  http://192.168.1.101:3000
```

> 服务默认监听 `0.0.0.0`，学生可通过局域网 IP 访问。

### 2.2 停止服务

- **Windows**：双击 `stop.bat`
- **macOS**：运行 `bash stop.sh` 或按 `Ctrl+C`

## 3. 教师端

### 3.1 首次配置

#### 首次启动的用户设置

首次启动时，浏览器会自动打开 `http://localhost:3000`，显示用户配置页面：

1. 填写**手机号**、**邮箱**、**姓名**、**学校/机构**
2. 设置**密码**（至少 6 位）和**确认密码**
3. 可选：设置**密码恢复问题**和答案
4. 点击「完成用户设置」

> **提示**：学校/机构名称需包含"学"、"园"、"院"、"中心"等关键词。

#### 密钥文件保存

注册成功后，系统会弹出**密钥文件**保存窗口：

- 密钥用于**忘记密码时恢复账号**
- **只能下载一次**，请务必妥善保管
- 点击「下载密钥文件」保存到本地
- 点击「已保存」进入教师工作台

#### 后续登录

已注册用户访问 `http://localhost:3000` 时，会自动跳转到登录页：

1. 输入手机号和密码
2. 点击「登录」
3. 登录后自动进入教师工作台

#### 忘记密码

在登录页面点击「忘记密码」，按提示通过**密保问题**重置密码。

---

### 3.2 创建课堂与班级管理

#### 创建课堂

1. 进入「课堂管理」页面
2. 点击「新建课堂」
3. 填写课堂信息：
   - **课堂标题**（必填）：如「一元一次方程」
   - **课堂目标**（必填）：描述本课堂的学习目标
   - **年级**（必填）：选择适用年级
   - **学科**（必填）：选择学科
   - **班级**（必填）：选择要分配的班级
4. 点击「保存」

创建成功后，课堂自动展开并进入对话活动编辑模式。

#### 班级管理

1. 进入「班级管理」页面
2. 可创建新班级，每个班级自动生成**邀请码**
3. 学生通过邀请码加入班级
4. 可设置是否开放邀请码注册（开启后学生可自助注册）

#### 课堂状态

| 状态 | 说明 |
|------|------|
| 已停用 | 课堂未发布，学生不可见 |
| 已启用 | 课堂已发布，学生可见可使用 |
| 已归档 | 课堂已结束，学生不可见 |

---

### 3.3 对话活动管理

对话活动是学生与 AI 进行主题对话的核心载体。

#### 编辑对话活动

1. 在课堂列表中展开目标课堂
2. 点击「活动管理」进入编辑模式
3. 可添加、编辑、删除对话活动

#### 对话活动字段说明

| 字段 | 说明 | 是否必填 |
|------|------|----------|
| 对话活动名称 | 活动标题，如「什么是方程？」 | 是 |
| 对话活动目标 | 活动描述和学习目标 | 是 |
| 对话提示词 | 注入 AI 的系统提示词，决定 AI 的角色和行为 | 是 |
| 个人学情分析提示词 | 生成学生个人分析时使用的提示词 | 是 |
| 全班学情分析提示词 | 生成全班分析时使用的提示词 | 是 |

#### AI 角色设计建议

对话提示词是 AI 在对话中的"行为准则"，建议遵循以下原则：

- **以问代答**：多通过提问引导学生思考，而非直接给出答案
- **鼓励质疑**：欢迎学生提出不同观点，培养批判性思维
- **联系生活**：适当联系学生熟悉的生活实例，帮助理解抽象概念
- **适时总结**：在学生理解关键知识点后，进行简明扼要的总结
- **分层引导**：根据学生的回答深度调整问题难度

#### 启用/禁用对话活动

- 在课堂列表中，每个对话活动旁有开关按钮
- 禁用后学生端不再显示该对话活动
- 已有的学生对话记录不会删除

#### 排序

- 使用上下箭头按钮调整对话活动顺序
- 学生端按此顺序显示

---

### 3.4 课堂作业管理

#### 创建作业

1. 在课堂展开区域，点击「作业管理」
2. 点击「新建作业」
3. 填写作业标题和描述

#### 添加题目

1. 进入作业详情，点击「添加题目」
2. 选择题型：单选题、多选题、填空题、简答题
3. 填写题目内容、选项（选择题）、答案、分值、难度
4. 也可使用 **AI 自动生成题目**

#### 作业状态

| 状态 | 说明 |
|------|------|
| 未生效 | 草稿状态，学生不可见。编辑保存后自动回到此状态 |
| 已生效 | 已发布，学生可答题。需至少添加一道题目才能生效 |

> **注意**：每次编辑保存作业后，状态会自动变为「未生效」，需重新发布。

#### 作业统计与报告

- **统计**：查看答题人数、平均分、正确率
- **报告**：AI 自动生成作业分析报告（详见"学情分析"章节）

---

### 3.5 互动探究管理

互动探究是 HTML 交互式学习活动，支持学生操作、提交和行为分析。

#### 创建探究活动

1. 在课堂展开区域，点击「互动探究」面板
2. 点击「新建探究」
3. 填写标题、描述
4. 编辑 HTML 交互内容（支持直接编辑 HTML 代码）

#### AI 生成题目

1. 编辑探究内容后，点击「AI 生成题目」
2. 系统根据探究内容自动生成 4 道题目（1 单选 + 1 多选 + 1 填空 + 1 简答）
3. 生成后可手动调整

> **注意**：已有学生提交后，不可修改探究内容和重新生成题目。

#### 提交功能

- 开启后学生可提交探究结果
- 已有提交后，内容和提交开关将被锁定

#### 行为分析

系统会自动记录学生在探究活动中的：
- 各类操作（点击、拖拽、输入）的频次与占比
- 提交时停留时长
- 答题尝试次数
- 互动行为模式

这些数据将用于学情分析报告。

### 3.6 对话记录查看与删除

1. 进入「对话记录」页面
2. 可按班级、课堂、对话活动筛选
3. 可按学生姓名搜索
4. 展开对话可查看完整聊天记录
5. 支持单个删除和批量删除
6. 支持清空某个班级的所有对话记录

### 3.7 系统配置

进入「系统设置」页面，包含以下配置：

#### AI 服务配置

| 配置项 | 说明 |
|--------|------|
| API Base URL | AI 服务地址，支持 OpenAI 兼容接口（通义千问、DeepSeek、智谱等） |
| API Key | API 密钥，仅存储在本地数据库 |
| AI 模型 | 模型名称，如 qwen-turbo、deepseek-chat |
| 推理思考模式 | 仅 DeepSeek V4 系列支持，返回推理过程 |
| AI 并发限制 | 同时处理的 AI 请求数，小班 10-20、中班 20-40、大班 40-100 |
| 测试连接 | 验证 AI 服务是否可用 |

> **推荐**：阿里云有许多免费的 AI 服务（如通义千问），可以使用，但需要申请阿里云账号。

#### 用户信息设置

- 修改单位（学校）
- 设置老师座右铭（显示在工作台）
- 设置学生座右铭（统一显示在所有学生端）
- 修改密码

#### 学情洞察配置

| 配置项 | 说明 |
|--------|------|
| 启动指数评价 | 开启后学生分析只输出星级评分（6-10 星），不显示文字 |
| 个人分析字数上限 | 建议 100-5000 字，留空不限 |
| 班级分析字数上限 | 建议 200-10000 字，留空不限 |

#### 配置备份与导入

- 可导出当前配置为 JSON 文件
- 可导入 JSON 配置文件恢复设置

## 4. 学情分析（核心功能）

学情分析是 QuickClass 的核心功能。系统基于学生的对话记录、作业答题和探究提交数据，由 AI 自动生成多维度分析报告。

### 4.1 学情分析矩阵

QuickClass 首创**"三类活动 × 两层视角"**的学情分析体系：

| 活动维度 | 学生个人分析 | 全班班级分析 |
|----------|--------------|--------------|
| **对话活动** | 个体对话表现分析 | 班级对话汇总分析 |
| **互动探究** | 个人探究行为分析 | 班级探究行为分析 |
| **课堂作业** | 个人答题情况分析 | 班级答题数据分析 |
| **综合课堂** | 学生个人综合分析 | 全班综合分析 |

**价值**：
- 横向看：从**活动维度**对比（哪个活动最有效）
- 纵向看：从**学生维度**追踪（某学生各活动表现）
- 整体看：从**班级维度**把握（找共性问题）

### 4.2 双报告模式

每类分析支持**两种输出形式**：

| 报告类型 | 特点 | 适用场景 |
|----------|------|----------|
| **文本型报告** | 精炼文字，结构清晰 | 快速浏览、复制粘贴、邮件发送 |
| **数据型（HTML）报告** | 含 ECharts 可视化图表（雷达图、柱状图、饼图、散点图） | 深度分析、投屏展示、打印存档 |

教师可在分析界面自由选择使用哪种报告。

### 4.3 进入学情分析

1. 在课堂列表中，点击课堂旁的「分析」按钮
2. 或进入「课堂管理」→ 展开课堂 → 点击具体活动的「分析」

---

### 4.4 对话活动分析

基于学生与 AI 的对话记录，分析学生对知识的理解程度。

#### 学生个人分析

- 选择具体学生，点击「生成分析」
- AI 根据该学生的**完整对话内容**进行分析：
  - 知识理解程度和薄弱点
  - 思维过程和表达特点
  - 学习态度与参与度
  - 个性化学习建议
- 支持批量一键分析所有学生
- 分析结果按星级指数排序，快速定位需要关注的学生

#### 全班分析

- 点击「生成班级分析」
- AI 汇总全班学生的对话数据，生成整体分析报告
- 报告内容：
  - **文本报告**：班级整体掌握情况、共性问题、教学建议
  - **HTML 大屏**：参与度饼图、学生活跃度柱状图、核心概念掌握统计
- 支持多版本管理，可对比不同时间点的分析结果

---

### 4.5 互动探究分析

基于学生的探究提交数据（操作记录、得分、用时），分析探究学习效果。

#### 班级分析

- 汇总全班探究提交数据，生成整体分析
- 报告内容：
  - **文本报告**：完成率、平均得分、得分分布、共性问题、教学建议
  - **HTML 大屏**：操作类型统计图、分数分布图、提交详情表
- **分析维度**：
  - 整体完成情况（参与度、完成率）
  - 学生行为特点（点击、拖拽、输入等操作占比）
  - 表现优秀和有困难的学生识别
  - 教学改进建议

#### 学生个人分析

- 查看学生探究提交详情：
  - 操作过程和交互记录
  - 得分和完成情况
  - AI 生成的个人学习建议
- 支持查看学生的操作日志（点击、滚动、输入等行为）

---

### 4.6 课堂作业分析

基于学生的作业答题数据，分析知识掌握情况。

#### 班级分析

- 汇总全班答题数据，生成整体分析
- 报告内容：
  - **文本报告**：正确率、平均分、得分分布、典型错误、教学建议
  - **HTML 大屏**：各题正确率柱状图、分数分布直方图、薄弱知识点分析
- **关键指标**：
  - 各题正确率（正确率<60% 自动标红）
  - 薄弱题目识别
  - 低分/高分学生分组
  - 班级整体掌握度评估

#### 学生个人分析

- 查看学生答题详情：
  - 各题答题情况和正误
  - 总分和各题型得分
  - AI 生成的错题分析和学习建议
- 支持查看学生的具体答案和批改结果

---

### 4.7 提示词模板系统

QuickClass 首创**模板驱动的 AI 分析系统**，教师可完全掌控 AI 分析的逻辑和输出。

#### 六类模板

| 模板类型 | 用途 | 默认模板数 |
|----------|------|------------|
| **对话设计** | 设计 AI 在对话活动中的角色和行为 | 1 个 |
| **学生个人学情** | 分析单个学生在学习活动中的表现 | 4 个（对话活动/课堂 × 文本型/数据型） |
| **学生全班学情** | 分析班级整体学习状态 | 4 个（对话活动/课堂 × 文本型/数据型） |
| **课堂作业设计** | AI 生成作业题目的提示词 | 1 个 |
| **课堂作业分析** | 分析作业答题数据 | 2 个（文本型/数据型） |
| **互动探究分析** | 分析互动探究提交结果 | 2 个（文本型/数据型） |

#### 模板管理

- 进入「提示词模板」页面管理所有模板
- 可创建、编辑、删除、复制模板
- 可设置**默认模板**（每类可设置一个）
- 支持**预览完整 Prompt**（含变量替换后的实际提示词）
- 支持 **AI 自动生成模板**（基于教师输入的需求）

#### 自定义模板的优势

- **可控性**：教师完全掌控 AI 分析的维度和侧重点
- **灵活性**：不同学科、不同教学场景可使用不同模板
- **复用性**：一次编写，多次使用，统一教学风格
- **数据化**：模板中可指定输出的图表类型和数据维度

#### 模板中的变量

模板支持以下变量（部分），教师可灵活组合：

| 变量 | 含义 |
|------|------|
| `{studentName}` | 学生姓名 |
| `{pcTitle}` | 对话活动名称 |
| `{pcDescription}` | 对话活动目标 |
| `{taskTitle}` | 课题 |
| `{taskGrade}` | 年级 |
| `{taskSubject}` | 学科 |
| `{taskDescription}` | 课堂目标 |
| `{personalDialogContents}` | 学生/班级对话内容 |
| `{personalQuizStats}` | 个人作业数据 |
| `{classQuizStats}` | 全班作业数据 |
| `{scoreDistribution}` | 分数分布 |
| `{actionTypeStats}` | 探究操作类型统计 |
| `{submissionDetails}` | 探究提交详情 |

#### 数据型报告的特殊设计

如果要让 AI 生成 HTML 可视化报告，模板中应包含以下关键词之一：
- `<!DOCTYPE`、`html`、`div`、`echarts`、`ECharts`、`chart`

系统会自动识别并切换到 HTML 输出模式（无字数限制），支持的图表类型：

- **ECharts 雷达图**：多维度能力对比
- **ECharts 柱状图**：各题正确率、活跃度对比
- **ECharts 饼图**：操作类型占比、参与度分布
- **ECharts 散点图**：学生分布
- **进度条**：知识点掌握度
- **表格**：详情数据展示

## 5. 学生端

### 5.1 登录与加入班级

#### 加入班级

1. 浏览器访问 `http://教师IP:3000/student`
2. 输入教师提供的**邀请码**
3. 输入姓名
4. 点击「加入」

系统会自动创建账号并加入对应班级。

#### 登录

- 已有账号的学生，输入姓名和密码登录
- 无密码的学生，输入姓名和邀请码登录

---

### 5.2 对话活动学习

1. 登录后进入对话页面
2. 左侧显示当前课堂和对话活动列表
3. 点击对话活动名称，开始与 AI 对话
4. 对话内容会被记录，教师可查看和分析

---

### 5.3 课堂作业答题

1. 在对话页面右侧，点击「课堂作业」标签
2. 选择要完成的作业
3. 逐题作答，提交后自动批改
4. 可查看得分和正确答案

---

### 5.4 互动探究

1. 在对话页面右侧，点击「互动探究」标签
2. 选择探究活动
3. 在 HTML 交互界面中完成学习任务
4. 点击「提交」提交探究结果

## 6. 常见问题

### Q: 学生看不到课堂？

检查以下几点：
1. 课堂状态是否为「已启用」
2. 课堂是否分配到了学生所在的班级
3. 对话活动是否已启用

### Q: AI 对话无响应？

1. 检查系统设置中的 AI 服务配置是否正确
2. 点击「测试连接」验证 AI 服务可用性
3. 检查 API Key 是否有效
4. 检查网络连接

### Q: 学情分析生成失败？

1. 确保学生已有对话记录或答题数据
2. 检查 AI 服务是否正常
3. 检查分析提示词是否为空
4. 确认已选择有效的分析模板

### Q: 如何切换文本型/数据型报告？

在分析页面，分析时使用的模板决定报告类型：
- 文本型模板 → 生成 Markdown 文本报告
- 数据型（HTML）模板 → 生成可视化 HTML 报告

可在「提示词模板」管理页面查看每个模板的输出类型。

### Q: 保存对话活动后，之前的对话记录丢失关联？

此问题已在最新版本中修复。保存对话活动时，系统会自动保留学生对话记录与对话活动的关联。

### Q: 如何重置数据库？

```bash
npx prisma db push --force-reset
npx tsx prisma/seed-full.ts
```

> **警告**：此操作会删除所有数据，不可恢复！先备份数据库文件 `prisma/dev.db`。

### Q: 如何更新软件版本？

请参考 **版本更新指南.md**，使用更新脚本一键完成：
- `update-windows.bat`
- `update-mac.sh`

## 7. 教学创新特点

QuickClass 的核心创新可以概括为：

> **从"经验驱动教学"到"数据驱动教学"**
> **从"一对多讲授"到"一人一案引导"**
> **从"结果评价"到"过程行为分析"**
> **从"AI 替代教师"到"AI 赋能教师"**

### 7.1 解决的核心痛点

- **个性化教学难以规模化**：AI 对话 + 自动批改，让每个学生都有"专属辅导"
- **作业批改与反馈滞后**：自动批改 + 即时反馈
- **学情分析缺乏数据支撑**：多维度自动分析，结构化报告
- **课堂互动形式单一**：对话活动 + 互动探究，主动参与
- **教学反思缺乏工具**：自动生成报告，可回溯可对比

### 7.2 全国首创特点

1. **"三类活动 × 两层视角"学情分析矩阵**：横纵结合，全面把握
2. **"文本型 + 数据型"双报告模式**：同一任务，两种输出
3. **模板驱动的 AI 分析系统**：教师掌控分析逻辑
4. **"AI 引导者"角色定位**：培养思维，而非获取答案
5. **从"结果分析"到"行为分析"**：理解"怎么知道"，而非只知"知不知道"
6. **"教师设计 + AI 执行"协同范式**：保留教师主导权
7. **完整本地化部署能力**：数据安全可控
