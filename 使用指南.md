# QuickClass 使用指南

## 1. 系统概述

QuickClass 是一个 AI 驱动的课堂教学辅助平台，帮助教师创建互动式学习活动、管理课堂作业、分析学情数据。核心功能包括：

- **对话活动**：学生与 AI 进行主题对话，深入理解知识点
- **课堂作业**：在线答题，自动批改，即时反馈
- **互动探究**：HTML 交互式学习活动，支持提交和评分
- **学情分析**：AI 自动生成个人和全班分析报告
- **知识库**：上传教学资料，AI 对话时自动引用

### 适用场景

- 课堂教学中的个性化辅导
- 课后作业布置与批改
- 学情数据采集与分析
- 探究式学习活动设计

## 2. 教师端

### 2.1 登录与账号管理

#### 登录

1. 浏览器访问 `http://localhost:3000`
2. 输入手机号和密码
3. 点击「登录」

首次登录后，系统会检查用户信息是否完整。如不完整，会提示前往设置页面补充。

#### 注册

1. 在登录页面点击「配置」
2. 填写手机号、密码、姓名
3. 可选填写邮箱、学校
4. 点击「配置」

#### 忘记密码

1. 在登录页面点击「忘记密码」
2. 按提示通过密保问题重置密码

---

### 2.2 创建课堂与班级管理

#### 创建课堂

1. 进入「课堂管理」页面
2. 点击「新建课堂」
3. 填写课堂信息：
   - **课堂标题**（必填）：如「一元一次方程」
   - **课堂目标**（必填）：描述本课堂的学习目标
   - **年级**（必填）：选择适用年级
   - **学科**（必填）：选择学科
   - **班级**（必填）：选择要分配的班级
4. 点击「保存」

创建成功后，课堂自动展开并进入对话活动编辑模式。

#### 班级管理

1. 进入「班级管理」页面
2. 可创建新班级，每个班级自动生成**邀请码**
3. 学生通过邀请码加入班级
4. 可设置是否开放邀请码注册（开启后学生可自助注册）

#### 课堂状态

| 状态 | 说明 |
|------|------|
| 已停用 | 课堂未发布，学生不可见 |
| 已启用 | 课堂已发布，学生可见可使用 |
| 已归档 | 课堂已结束，学生不可见 |

---

### 2.3 对话活动管理

对话活动是学生与 AI 进行主题对话的核心载体。

#### 编辑对话活动

1. 在课堂列表中展开目标课堂
2. 点击「活动管理」进入编辑模式
3. 可添加、编辑、删除对话活动

#### 对话活动字段说明

| 字段 | 说明 | 是否必填 |
|------|------|----------|
| 对话活动名称 | 活动标题，如「什么是方程？」 | 是 |
| 对话活动目标 | 活动描述和学习目标 | 是 |
| 对话提示词 | 注入 AI 的系统提示词，决定 AI 的角色和行为 | 是 |
| 个人学情分析提示词 | 生成学生个人分析时使用的提示词 | 是 |
| 全班学情分析提示词 | 生成全班分析时使用的提示词 | 是 |

#### 启用/禁用对话活动

- 在课堂列表中，每个对话活动旁有开关按钮
- 禁用后学生端不再显示该对话活动
- 已有的学生对话记录不会删除

#### 排序

- 使用上下箭头按钮调整对话活动顺序
- 学生端按此顺序显示

---

### 2.4 课堂作业管理

#### 创建作业

1. 在课堂展开区域，点击「作业管理」
2. 点击「新建作业」
3. 填写作业标题和描述

#### 添加题目

1. 进入作业详情，点击「添加题目」
2. 选择题型：单选题、多选题、填空题、简答题
3. 填写题目内容、选项（选择题）、答案、分值、难度
4. 也可使用 AI 自动生成题目

#### 作业状态

| 状态 | 说明 |
|------|------|
| 未生效 | 草稿状态，学生不可见。编辑保存后自动回到此状态 |
| 已生效 | 已发布，学生可答题。需至少添加一道题目才能生效 |

> **注意**：每次编辑保存作业后，状态会自动变为「未生效」，需重新发布。

#### 作业统计与报告

- **统计**：查看答题人数、平均分、正确率
- **报告**：AI 自动生成作业分析报告

---

### 2.5 互动探究管理

互动探究是 HTML 交互式学习活动，支持学生操作和提交。

#### 创建探究活动

1. 在课堂展开区域，点击「互动探究」面板
2. 点击「新建探究」
3. 填写标题、描述
4. 编辑 HTML 交互内容（支持直接编辑 HTML 代码）

#### AI 生成题目

1. 编辑探究内容后，点击「AI 生成题目」
2. 系统根据探究内容自动生成 4 道题目（1 单选 + 1 多选 + 1 填空 + 1 简答）
3. 生成后可手动调整

> **注意**：已有学生提交后，不可修改探究内容和重新生成题目。

#### 提交功能

- 开启后学生可提交探究结果
- 已有提交后，内容和提交开关将被锁定

---

### 2.6 对话记录查看与删除

1. 进入「对话记录」页面
2. 可按班级、课堂、对话活动筛选
3. 可按学生姓名搜索
4. 展开对话可查看完整聊天记录
5. 支持单个删除和批量删除
6. 支持清空某个班级的所有对话记录

---

### 2.7 系统配置

进入「系统设置」页面，包含以下配置：

#### AI 服务配置

| 配置项 | 说明 |
|--------|------|
| API Base URL | AI 服务地址，支持 OpenAI 兼容接口（通义千问、DeepSeek、智谱等） |
| API Key | API 密钥，仅存储在本地数据库 |
| AI 模型 | 模型名称，如 qwen-turbo、deepseek-chat |
| 推理思考模式 | 仅 DeepSeek V4 系列支持，返回推理过程 |
| AI 并发限制 | 同时处理的 AI 请求数，小班 10-20、中班 20-40、大班 40-100 |
| 测试连接 | 验证 AI 服务是否可用 |
===阿里云有许多免费的AI服务，可以使用，但是需要申请阿里云账号，申请后可以使用===

#### 用户信息设置

- 修改单位（学校）
- 设置老师座右铭（显示在工作台）
- 设置学生座右铭（统一显示在所有学生端）
- 修改密码

#### 学情洞察配置

| 配置项 | 说明 |
|--------|------|
| 启动指数评价 | 开启后学生分析只输出星级评分（6-10 星），不显示文字 |
| 个人分析字数上限 | 建议 100-5000 字，留空不限 |
| 班级分析字数上限 | 建议 200-10000 字，留空不限 |

#### 配置备份与导入

- 可导出当前配置为 JSON 文件
- 可导入 JSON 配置文件恢复设置

### 2.8 学情分析

学情分析是 QuickClass 的核心功能，基于学生的对话记录、作业答题和探究提交数据，AI 自动生成分析报告。分析分为**个人分析**和**全班分析**两个维度。

#### 进入学情分析

1. 在课堂列表中，点击课堂旁的「分析」按钮
2. 或进入「课堂管理」→ 展开课堂 → 点击对话活动旁的「分析」

#### 分析层级

- **对话活动层级**：选择具体对话活动，查看该活动下的分析，
- **课堂层级**：查看整个课堂的综合分析

---

#### 对话学情分析

基于学生与 AI 的对话记录，分析学生对知识的理解程度。

**个人分析**

- 选择具体学生，点击「生成分析」
- AI 根据该学生的对话内容，分析：
  - 知识理解程度和薄弱点
  - 思维过程和表达特点
  - 学习建议和改进方向
- 支持批量一键分析所有学生
- 分析结果按星级指数排序，快速定位需要关注的学生

**全班分析**

- 点击「生成班级分析」
- AI 汇总全班学生的对话数据，生成整体分析报告
- 报告包含两种呈现形式：
  - **文本分析**：结构化的文字报告，包含班级整体掌握情况、共性问题、教学建议
  - **HTML 大屏分析**：可视化数据大屏，适合课堂投屏展示，包含：
    - 班级整体掌握度雷达图
    - 各知识点理解程度柱状图
    - 学生分布散点图
    - 关键指标统计卡片
- 支持多版本管理，可对比不同时间点的分析结果

---

#### 互动探究分析

基于学生的探究提交数据（操作记录、得分、用时等），分析探究学习效果。

**个人分析**

- 选择具体学生，查看其探究提交详情：
  - 操作过程和交互记录
  - 得分和完成情况
  - AI 生成的个人学习建议
- 支持查看学生的操作日志（点击、滚动、输入等行为）

**全班分析**

- 汇总全班探究提交数据，生成整体分析
- 报告包含两种呈现形式：
  - **文本分析**：全班探究完成率、平均得分、得分分布、共性问题、教学建议
  - **HTML 大屏分析**：可视化展示：
    - 完成率和参与度统计
    - 得分分布直方图
    - 各探究环节完成情况热力图
    - 典型问题和高频错误分析
- 支持多版本管理和版本对比

---

#### 作业分析

基于学生的作业答题数据，分析知识掌握情况。

**个人分析**

- 选择具体学生，查看其答题详情：
  - 各题答题情况和正误
  - 总分和各题型得分
  - AI 生成的错题分析和学习建议
- 支持查看学生的具体答案和批改结果

**全班分析**

- 汇总全班答题数据，生成整体分析
- 报告包含两种呈现形式：
  - **文本分析**：全班正确率、平均分、得分分布、典型错误、教学建议
  - **HTML 大屏分析**：可视化展示：
    - 各题正确率柱状图
    - 得分分布直方图
    - 选项选择分布饼图
    - 高频错误和知识点薄弱环节分析
- 支持多版本管理和版本对比

---

#### 分析提示词模板

- 系统内置多种分析提示词模板
- 教师可自定义模板，控制分析报告的格式和侧重点
- 支持 AI 自动生成提示词
- 发送前可预览完整 Prompt，确认后执行

## 3. 学生端

### 3.1 登录与加入班级

#### 加入班级

1. 浏览器访问 `http://教师IP:3000/student`
2. 输入教师提供的**邀请码**
3. 输入姓名
4. 点击「加入」

系统会自动创建账号并加入对应班级。

#### 登录

- 已有账号的学生，输入姓名和密码登录
- 无密码的学生，输入姓名和邀请码登录

---

### 3.2 对话活动学习

1. 登录后进入对话页面
2. 左侧显示当前课堂和对话活动列表
3. 点击对话活动名称，开始与 AI 对话
4. 对话内容会被记录，教师可查看和分析

---

### 3.3 课堂作业答题

1. 在对话页面右侧，点击「课堂作业」标签
2. 选择要完成的作业
3. 逐题作答，提交后自动批改
4. 可查看得分和正确答案

---

### 3.4 互动探究

1. 在对话页面右侧，点击「互动探究」标签
2. 选择探究活动
3. 在 HTML 交互界面中完成学习任务
4. 点击「提交」提交探究结果

## 4. 常见问题

### Q: 学生看不到课堂？

检查以下几点：
1. 课堂状态是否为「已启用」
2. 课堂是否分配到了学生所在的班级
3. 对话活动是否已启用

### Q: AI 对话无响应？

1. 检查系统设置中的 AI 服务配置是否正确
2. 点击「测试连接」验证 AI 服务可用性
3. 检查 API Key 是否有效
4. 检查网络连接

### Q: 学情分析生成失败？

1. 确保学生已有对话记录或答题数据
2. 检查 AI 服务是否正常
3. 检查分析提示词是否为空

### Q: 保存对话活动后，之前的对话记录丢失关联？

此问题已在最新版本中修复。保存对话活动时，系统会自动保留学生对话记录与对话活动的关联。

### Q: 如何重置数据库？

```bash
npx prisma db push --force-reset
npx tsx prisma/seed-full.ts
```

> **警告**：此操作会删除所有数据，不可恢复！先备份数据库文件 /prisma/dev.db
