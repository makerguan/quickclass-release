# QuickClass 版本更新指南

## 更新前准备

### 1. 备份确认

更新前请确认以下内容已备份：

| 备份项 | 位置 | 说明 |
|--------|------|------|
| 数据库 | `prisma/dev.db` | 包含所有班级、学生、对话等数据 |
| 自定义模板 | 数据库内 | 教师自建的模板存储在数据库中，随 `dev.db` 一起保留 |

> **注意**：`.env` 文件中的配置（如 API Key）在更新时会被覆盖，请提前记录。

### 2. 下载新版

从发布渠道下载最新的"跨平台安装包.zip"。

---

## 更新方式

### 方式一：自动更新脚本（推荐）

#### 步骤

1. 将下载的"跨平台安装包.zip"解压
2. 将解压后的文件夹放在**旧版目录的同级**，目录结构如下：

```
父目录/
  QuickClass/           ← 旧版（正在使用的）
  跨平台安装包/          ← 新版（刚解压的）
```

3. 在旧版目录中运行更新脚本：

**Windows**：双击 `update-windows.bat`

**macOS**：打开终端执行
```bash
cd 旧版QuickClass目录
bash update-mac.sh
```

4. 脚本会自动完成以下操作：

| 步骤 | 操作 | 说明 |
|------|------|------|
| ① | 备份数据库 | 复制 `dev.db` → `dev.db.bak` |
| ② | 同步新版文件 | 排除数据库、缓存、依赖 |
| ③ | 恢复数据库 | 将备份的数据库放回原位 |
| ④ | 更新数据库结构 | `prisma migrate deploy`（安全迁移，不丢数据） |
| ⑤ | 重新构建 | `npm run build` |

5. 更新完成后，使用启动脚本启动即可

---

### 方式二：手动更新

如果自动脚本无法使用，可手动操作：

#### 步骤

1. **备份数据库**

```bash
# 复制数据库文件到安全位置
cp prisma/dev.db ~/dev.db.bak
```

2. **记录环境配置**

打开 `.env` 文件，记录其中的 API Key 等自定义配置。

3. **用新版覆盖旧版**

将新版的所有文件复制到旧版目录，覆盖同名文件。

4. **恢复数据库**

```bash
# 将备份的数据库放回
cp ~/dev.db.bak prisma/dev.db
```

5. **恢复环境配置**

将之前记录的 API Key 等配置写回 `.env` 文件。

6. **安装依赖并更新**

```bash
npm install
npx prisma generate
npx prisma migrate deploy
npm run build
```

7. **启动服务**

```bash
# Windows
start-windows.bat

# macOS
bash start-mac.sh
```

## 常见问题

### Q: 更新后数据会丢失吗？

不会。更新脚本会自动备份数据库，且 `prisma migrate deploy` 只更新数据库结构，不会删除数据。

### Q: 更新后启动报错怎么办？

1. 确认 `prisma/dev.db` 文件存在
2. 尝试重新执行：
```bash
npx prisma generate
npx prisma migrate deploy
npm run build
```
3. 如果仍然报错，检查 `.env` 文件中的配置是否正确

### Q: 如何回退到旧版本？

1. 从 GitHub 下载旧版代码
2. 恢复备份的 `dev.db` 文件
3. 执行 `npm install && npx prisma generate && npm run build`

### Q: 更新时提示"prisma migrate deploy"失败？

可能是数据库结构与迁移记录不一致，尝试：
```bash
npx prisma migrate reset
```
> **警告**：此操作会清空数据库，仅在其他方式无效时使用，并确保已备份 `dev.db`。

### Q: .env 文件被覆盖了怎么办？

新版 `.env` 可能包含新的配置项。请对比新旧 `.env`，将你之前的 API Key 等配置填入新版 `.env` 中。
